# AI Student Academic Advisor — Prototype

A LangGraph multi-agent prototype of the Lean Canvas concept: five agents
that turn a student's grades and interests into a performance report,
study plan, resource list, and career suggestions.

```
student_profile → performance_analysis → study_planner → resource_recommendation → career_guidance
                        ^   |
                        └───┘ (loops back if no grade data yet, capped retries)
```

## Agents

| Agent | File | Responsibility |
|---|---|---|
| Student Profile Agent | `src/agents/student_profile_agent.py` | Loads program/year/interests, summarizes them |
| Performance Analysis Agent | `src/agents/performance_analysis_agent.py` | Computes GPA, flags weak/strong subjects |
| Study Planner Agent | `src/agents/study_planner_agent.py` | Builds a 4-week, hour-allocated study plan |
| Resource Recommendation Agent | `src/agents/resource_recommendation_agent.py` | Matches open resources to weak subjects |
| Career Guidance Agent | `src/agents/career_guidance_agent.py` | Ranks careers by subject/interest overlap |

## Stack

- **[LangGraph](https://github.com/langchain-ai/langgraph)** — agent orchestration (open source)
- **[Ollama](https://ollama.com)** via `langchain-ollama` — local, open-source LLM (llama3.1 etc.), no API key. Falls back automatically to a deterministic mock LLM if Ollama isn't running, so the prototype always runs.
- **SQLite** (stdlib) — student/grades/resources/careers storage
- **Rich** — CLI output formatting

## Quickstart

```bash
python setup.py            # installs deps, creates .env, seeds the SQLite DB
python main.py             # interactive: type in your own profile + grades
```

`setup.py` still seeds a demo database, and the original lookup-by-id mode
is kept for backwards compatibility / demoing without typing anything in:

```bash
python main.py --student S001
python main.py --student S002
```

### Two input modes

- **Interactive (default, `python main.py` or `--interactive`)** — you're
  prompted for your name, program, year, and interests, then your grades
  one subject at a time. Nothing personal is read from the database; the
  Student Profile and Performance Analysis agents use exactly what you
  typed in.
- **Database demo mode (`--student <id>`)** — the original behavior: looks
  up a pre-seeded student's profile and grades from `data/advisor.db`.

In both modes, the resource and career **catalogs** (not personal data)
still come from the database, since those are shared reference data every
student draws from.

To use a real open-source model instead of the mock fallback:

```bash
# in a separate terminal
ollama pull llama3.1
ollama serve
```

`setup.py` checks for Ollama automatically and tells you which mode you're in.

## Project layout

```
setup.py                 # bootstrap: install deps, seed DB, check Ollama
cleanup.py                # remove generated DB/.env/caches
main.py                   # CLI entry point
requirements.txt
.env.example
data/
  seed_data.json          # sample students, grades, resources, careers
  advisor.db              # generated by setup.py (git-ignored)
src/
  llm.py                  # pluggable LLM (Ollama / mock)
  state.py                # shared LangGraph state
  db.py                   # SQLite access layer
  graph.py                # StateGraph wiring
  agents/                 # the five agents
tests/
  test_graph.py
```

## Notes / next steps for a real product

- Swap SQLite for a managed Postgres and add write paths (uploading transcripts, syllabus ingestion).
- Add a Q&A node (subject-related questions) using retrieval over course material.
- Add auth and per-institution data isolation before any real student data touches this.
- The mock LLM exists purely so this prototype runs with zero setup — replace it by pointing `.env` at Ollama (or any LangChain-supported provider) for real generated text.
